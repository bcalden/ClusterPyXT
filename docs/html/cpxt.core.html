This function calls the get_filenames_matching function and returns the first result. Useful when only one file is expected and you want to deal with that Path object not figure out if you got a list returned and then having to figure out which index to use. This alleviates that concern.

cpxt.core.io.get_filenames_matching(pattern: str) → list

This function uses the glob library to find filenames matching the pattern provided. Simple wild cards such as an asterix, *, can be used to match file names you only know pieces of. Types of patterns that can be used can be found here https://docs.python.org/3/library/glob.html .

patternstr
The file pattern we are trying to match. For example:

..ching(‘/path/to/cluster/obs/analysis/acis_ccd[0-3].fits’)

would return a list of the 4 ‘acis_ccd#.fits’ files for that particular observation.

list

A list of Path objects pointing to the matching files.
